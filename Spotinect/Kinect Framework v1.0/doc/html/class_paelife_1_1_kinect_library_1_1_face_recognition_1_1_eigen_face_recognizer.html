Detailed Description

An implementation of IFaceRecognizer that uses the Eigen Faces and PCA (Principle Component Analysis) to reconize the faces. It only uses Kinect color frame data.

void Paelife.KinectLibrary.FaceRecognition.EigenFaceRecognizer.DetectAndSaveFace ( string  username)

It access the current frame of Kinect, detects and saves the face in it. To keep the Face Recognition as simple as it can be, only one person can be in the captured Kinect area. [see details bellow]

Therefore, this method can throw a FaceRecognitionException. Check the FaceRecognitionException.Message for details on the exception. It can be one of the following:

  • Could not read the current frame from Kinect.
  • Unable to detect a face.
  • More than one face was detected.
  • IO exceptions related to saving the image in the disk.

double Paelife.KinectLibrary.FaceRecognition.EigenFaceRecognizer.TresholdLevel
getset

It is initialized with a value of -1, which means that the recognizer will always return a username when recognizing a face, despite the differences between the faces. [see details bellow]

If you want a good commitment between accuracy and successful recognition, a value around 3000 is recommended. Bellow 3000, means that the Face Recognizer will be more accurate, but there will be more times when a user can't be recognized. If a value over 3000 is provided, it will recognize users more often, but with less accuracy.
